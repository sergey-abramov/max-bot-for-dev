Max Bot — бот для MaxHub с викториной и чатом с ИИ
==================================================

Небольшой пример бота на `@maxhub/max-bot-api` с:
- главным меню,
- викториной по Java,
- чатом с ИИ (OpenRouter),
- поддержкой голосовых сообщений через локальный STT‑сервис на faster‑whisper.

Как запустить
-------------

1. Клонируйте репозиторий и перейдите в папку проекта:

   ```bash
   git clone git@github.com:sergey-abramov/max-bot.git
   cd max-bot
   ```

2. Создайте `.env` на основе `.env.example` и заполните переменные:

   ```bash
   cp .env.example .env
   # отредактируйте .env и задайте:
   # BOT_TOKEN, OPENROUTER_API_KEY, OPENROUTER_MODEL и при желании STT_* переменные
   ```

3. Соберите и запустите сервисы через Docker Compose:

   ```bash
   docker-compose up --build
   ```

4. После успешного старта:
   - откройте чат с ботом в MaxHub;
   - отправьте `/start`;
   - выберите нужный пункт меню (информация, викторина, чат с ИИ).

Основной функционал
-------------------

1. **Главное меню**
   - Доступно через команду `/start`.
   - Бот отправляет сообщение с инлайн‑клавиатурой:
     - **«Информация обо мне»** — показывает данные текущего пользователя.
     - **«Викторина по Java»** — запускает мини‑тест по Java.
     - **«Чат с ИИ»** — включает режим общения с нейросетью через OpenRouter.

2. **Информация о пользователе**
   - По нажатию кнопки **«Информация обо мне»** бот отправляет:
     - ID пользователя;
     - имя;
     - username (если есть);
     - роль.
   - Все действия логируются в консоль с подробной информацией о пользователе и чате.

3. **Викторина по Java**
   - Запускается по кнопке **«Викторина по Java»** из главного меню.
   - Содержит несколько статических вопросов по основам Java и ООП.
   - Каждый вопрос отправляется вместе с инлайн‑кнопками с вариантами ответов (1–4).
   - Принимаются только ответы через кнопки:
     - любые текстовые сообщения во время викторины считаются "не по теме", бот просит пользоваться кнопками;
     - команды во время викторины также блокируются и сопровождаются подсказкой.
   - В конце викторины бот показывает итоговый результат вида:  
     `Тест завершён! Ваш результат: X из N правильных ответов.`
   - Старт и каждый ответ логируются в консоль с указанием пользователя, payload и текущего состояния викторины.

4. **Чат с ИИ (OpenRouter)**
   - Запускается по кнопке **«Чат с ИИ»** из главного меню.
   - Включает режим, в котором все обычные сообщения пользователя отправляются в выбранную модель через OpenRouter.
   - Поддерживает:
     - **текстовые вопросы** — сразу уходят в модель;
     - **голосовые сообщения** — сначала отправляются в локальный сервис распознавания речи (STT на faster‑whisper), затем распознанный текст уходит в модель.
   - Использует официальный SDK `@openrouter/sdk` и одну выбранную модель (по умолчанию `qwen/qwen-2.5-7b-instruct`).
   - При получении команды (`/start` и др.) режим чата с ИИ автоматически отключается.

5. **Обработка ошибок**
   - Большинство хэндлеров (`/start`, кнопки меню, викторина, чат с ИИ) обёрнуты в `try/catch` с логированием ошибок и понятными ответами пользователю.
   - Настроен глобальный обработчик ошибок `bot.catch`, который:
     - логирует все необработанные исключения (со стеком и типом обновления);
     - пытается отправить пользователю сообщение:  
       «Произошла непредвиденная ошибка. Попробуйте позже.»

Структура проекта
-----------------

- `src/bot.js` — основной файл бота:
  - инициализация `Bot`;
  - регистрация команды `/start`;
  - обработчики инлайн‑кнопок главного меню;
  - глобальный обработчик сообщений `message_created`;
  - глобальная обработка ошибок `bot.catch`.

- `src/handlers/victorine.js` — хэндлеры викторины:
  - статический набор вопросов и ответов;
  - хранение состояния викторины по пользователю;
  - отправка вопросов с инлайн‑клавиатурой;
  - обработка ответов по `message_callback`.

- `src/handlers/aiChat.js` — хэндлеры чата с ИИ:
  - управление режимом чата с ИИ по пользователю;
  - интеграция с OpenRouter через официальный SDK;
  - обработка текстовых и голосовых сообщений (распознавание речи + запрос к ИИ).

Работа с БД викторины
---------------------

Для хранения тем, вопросов и статистики по ответам используется PostgreSQL + SQLAlchemy + Alembic.

- **Подключение к БД**
  - Строка подключения передается через переменную окружения `DATABASE_URL`.
  - Формат: `postgresql+psycopg://user:password@host:5432/dbname`.
  - В Docker‑режиме обычно это что‑то вроде:  
    `DATABASE_URL=postgresql+psycopg://maxbot:maxbot@db:5432/maxbot`.

- **Миграции Alembic**
  - Перед первым запуском БД примените миграции:

    ```bash
    # из корня проекта
    alembic upgrade head
    ```

- **Начальное наполнение (сидинг)**
  - Для создания базовых тем и вопросов есть скрипт `db/seed.py`.
  - Убедитесь, что `DATABASE_URL` указывает на нужную БД, и выполните:

    ```bash
    python -m db.seed
    # или
    python db/seed.py
    ```

- **Тестовая БД**
  - Для изоляции тестов рекомендуется создать отдельную БД, например `maxbot_test`.
  - Эту БД можно указать:
    - либо через `TEST_DATABASE_URL` (предпочтительно для тестов);
    - либо через `DATABASE_URL`, если отдельная переменная для тестов не используется.

Запуск через Docker Compose
---------------------------

1. Создайте файл `.env` в корне проекта и задайте в нём необходимые переменные:

   ```bash
   BOT_TOKEN=ваш_токен_бота
   PORT=3000

   # Ключ и модель OpenRouter для чата с ИИ
   OPENROUTER_API_KEY=your_openrouter_api_key_here
   OPENROUTER_MODEL=qwen/qwen-2.5-7b-instruct

   # Настройки STT (распознавание речи, например OpenAI Whisper)
   STT_API_KEY=your_stt_api_key_here
   STT_API_URL=https://api.openai.com/v1/audio/transcriptions
   STT_MODEL=whisper-1
   ```

2. Соберите и запустите контейнер:

   ```bash
   docker-compose up --build
   ```

3. После успешного запуска сервис `bot` будет автоматически получать обновления от MaxHub и обрабатывать:
   - команду `/start`;
   - главное меню (информация о пользователе, викторина, чат с ИИ);
   - сообщения в активной викторине или режиме чата с ИИ.

Тесты для модуля БД
-------------------

Для базовой проверки работы ORM‑моделей и репозиториев используются тесты на `pytest` в папке `tests/`.

- **Предварительные требования**
  - Установите зависимости для Python‑части проекта (включая `pytest`).
  - Поднимите PostgreSQL и создайте отдельную БД для тестов (например, `maxbot_test`).
  - Установите одну из переменных окружения:
    - `TEST_DATABASE_URL` — отдельная строка подключения только для тестов;
    - или `DATABASE_URL` — если хотите использовать одну и ту же БД для приложения и тестов.

- **Запуск тестов**

  ```bash
  # Вариант с отдельной тестовой БД
  export TEST_DATABASE_URL="postgresql+psycopg://maxbot:maxbot@localhost:5432/maxbot_test"
  # или, если отдельная тестовая БД не используется
  # export DATABASE_URL="postgresql+psycopg://maxbot:maxbot@localhost:5432/maxbot"
  pytest -q
  ```

- **Запуск только Python‑части (БД + сервис викторины)**
  - Для разработки и отладки слоя БД и Python‑сервисов необязательно поднимать JS‑бот:
    1. Установите Python‑зависимости проекта.
    2. Поднимите PostgreSQL и задайте `DATABASE_URL` (и при желании `TEST_DATABASE_URL`).
    3. Примените миграции и, при необходимости, выполните сидинг.
    4. Запускайте `pytest`, чтобы прогонять только тесты `tests/`, либо поднимайте FastAPI‑приложение из `services/quiz_api.py` (например, через `uvicorn services.quiz_api:app`).

Тесты:

- проверяют создание и обновление пользователя (`db/repositories/user_repo.py`);
- проверяют получение тем и вопросов, включая случайный выбор (`db/repositories/topic_repo.py`, `db/repositories/question_repo.py`);
- проверяют запись и агрегацию статистики по темам (`db/repositories/stat_repo.py`).
